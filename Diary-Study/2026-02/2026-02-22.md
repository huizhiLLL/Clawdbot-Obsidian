# 学习笔记 - 2026-02-22

## 主题：Git 基础命令

### 核心知识点

#### 1. init - 初始化仓库
```bash
git init
```
- 在当前目录创建一个新的 Git 仓库
- 会创建一个隐藏的 `.git` 目录，存储所有版本控制信息

#### 2. add - 添加文件到暂存区
```bash
git add <文件名>          # 添加单个文件
git add .                 # 添加当前目录所有文件
git add *.js              # 添加所有 .js 文件
```
- 将文件修改添加到暂存区（staging area）
- 暂存区是提交前的准备区

#### 3. commit - 提交更改
```bash
git commit -m "提交信息"
```
- 将暂存区的内容提交到仓库历史
- `-m` 后跟提交信息，描述这次更改的内容
- 提交信息应简洁明了

#### 4. push - 推送到远程仓库
```bash
git push                  # 推送到默认远程分支
git push origin main      # 推送到 origin 远程仓库的 main 分支
```
- 将本地提交推送到远程仓库（如 GitHub、GitLab）
- 首次推送可能需要设置上游分支：`git push -u origin main`

#### 5. pull - 拉取远程更新
```bash
git pull                  # 拉取并合并远程更新
git pull origin main      # 从指定远程分支拉取
```
- 从远程仓库获取最新代码并合并到本地
- 相当于 `git fetch` + `git merge`

#### 6. branch - 分支管理
```bash
git branch                # 列出所有分支
git branch <分支名>       # 创建新分支
git branch -d <分支名>    # 删除已合并的分支
git checkout <分支名>     # 切换到指定分支
git checkout -b <分支名>  # 创建并切换到新分支
```
- 分支允许并行开发，互不影响
- `main` 或 `master` 通常是主分支

### 常用场景

#### 场景1：首次提交代码
```bash
git init                      # 初始化
git add .                     # 添加所有文件
git commit -m "Initial commit" # 首次提交
git remote add origin <URL>   # 添加远程仓库
git push -u origin main       # 推送并设置上游
```

#### 场景2：日常开发流程
```bash
git add .                     # 添加修改
git commit -m "Fix login bug" # 提交
git push                      # 推送到远程
```

#### 场景3：多人协作
```bash
git pull                      # 先拉取他人更改
# 解决冲突后
git add .
git commit -m "Resolve conflicts"
git push
```

### 最佳实践

1. **提交信息规范**：
   - 使用动词开头：`Add`、`Fix`、`Update`、`Remove`
   - 简短描述，不超过 50 字符
   - 示例：`Add user authentication`、`Fix navbar alignment`

2. **小步提交**：
   - 频繁提交，每次提交一个功能或修复
   - 便于回滚和代码审查

3. **提交前检查**：
   ```bash
   git status                  # 查看当前状态
   git diff                    # 查看未暂存的修改
   git diff --staged           # 查看暂存区的修改
   ```

4. **忽略不需要的文件**：
   - 创建 `.gitignore` 文件
   - 忽略 `node_modules`、`.env`、日志文件等

### 关联知识点

- **分布式版本控制**：Git 每个克隆都是完整的仓库
- **SHA-1 哈希**：每次提交有唯一的哈希值
- **HEAD**：指向当前分支的最新提交
- **暂存区**：提交前的缓冲区，允许选择性提交
